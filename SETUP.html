🧱 ForteBase ERP · 傻瓜部署指南

从零开始，大约 15 分钟，完全免费

1注册 GitHub（代码托管，免费）
→ 点这里注册 GitHub

随便填邮箱和用户名，一路点 Continue，验证邮箱就完成了。

💡 已有GitHub账号？直接跳到第2步
2在 GitHub 新建仓库
→ 新建仓库

• Repository name：填 fortebase-erp（随便起）
• 选 Public（公开，Vercel免费版需要）
• 勾选 Add a README file
• 点 Create repository

3上传这个文件夹里的所有文件

在刚才新建的仓库页面：

1. 点 Add file → Upload files
2. 把这个 deploy 文件夹里的所有内容（不是文件夹本身）拖进去：

📄 index.html
📄 manifest.json
📄 sw.js
📄 vercel.json
📁 icons/
  ├── icon-72.png
  ├── icon-96.png
  ├── icon-128.png
  ├── icon-192.png
  └── icon-512.png

3. 拖完点 Commit changes

💡 icons 文件夹需要进入文件夹后再上传里面的文件，或者直接把5个png全部拖进去（GitHub会自动识别路径）
4注册 Vercel 并部署（免费托管）
→ 点这里注册 Vercel

1. 选 Continue with GitHub（用GitHub账号登录）
2. 点 Add New → Project
3. 找到 fortebase-erp 仓库 → 点 Import
4. 什么都不用改 → 直接点 Deploy
5. 等1分钟，出现 Congratulations!

✅ 部署成功！你会得到一个网址，类似：
https://fortebase-erp.vercel.app
5配置 Supabase 云同步（免费）
→ 注册 Supabase

1. 注册后点 New project
2. Organization 选默认，填项目名 fortebase-erp
3. Database Password 设一个强密码（保存好）
4. Region 选 Southeast Asia (Singapore)（安哥拉最近）
5. 点 Create new project，等约2分钟

建表（运行一次）：

进入 SQL Editor，复制粘贴以下内容，点 Run：

CREATE TABLE IF NOT EXISTS erp_snapshots ( device_id TEXT PRIMARY KEY, snapshot JSONB NOT NULL, saved_at TIMESTAMPTZ DEFAULT now(), version TEXT, store_count INT DEFAULT 1, store_names TEXT, boss_name TEXT, sale_count INT DEFAULT 0, product_count INT DEFAULT 0 ); ALTER TABLE erp_snapshots ENABLE ROW LEVEL SECURITY; CREATE POLICY "allow_all" ON erp_snapshots FOR ALL USING (true) WITH CHECK (true);

获取密钥：

左侧菜单 Settings → API，复制：
• Project URL（如 https://xxxx.supabase.co）
• anon public key（很长的字符串）

6在APP里填入云端配置

1. 手机/电脑打开你的 Vercel 网址
2. 用老板PIN登录
3. 点左侧 ⚙️ 设置
4. 切换到 💾 数据备份 标签
5. 找到 ☁️ 云端同步 区域
6. 填入第5步复制的 URL 和 Key
7. 点 保存并连接

✅ 右上角出现 ☁️ 已同步 = 全部完成！
7安装到安卓手机桌面

1. 手机 Chrome 打开你的网址
2. 屏幕下方会出现橙色安装横幅 → 点 安装
3. 或者：Chrome 右上角 ⋮ → 添加到主屏幕
4. 桌面出现 🧱 图标，打开就是APP，全屏无地址栏

💡 4台设备（2台POS电脑 + 2台老板手机）都用浏览器打开同一个网址，各自安装或书签。数据通过云端自动同步。
🎉
完成！

多台设备自动同步的ForteBase ERP系统
